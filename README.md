## 결론 및 시사점

### 1. 현재 네이밍의 공통 패턴 (depth 3 이상)

depth가 3 이상인 경우, 아래 순서가 **가장 빈도 높은 패턴**입니다:

```
depth_1: {위치정보}        → 6단배너, 메인1단, 텍스트배너 등
depth_2: {소재/제휴사}     → 마케팅, 중개, LG전자제휴 등  
depth_3: {상세내용}        → 상품목록, 기간, 브랜드 등
depth_N(마지막): {노출대상} → 그룹사/오픈사, 탄생응원몰, LG전자/전자외 등
```

### 2. `{프로모션}_{00단}_{000배너}` 표준화 검토 결과

| 제안 패턴 | 현실 | 판정 |
|:---|:---|:---:|
| depth_1 = `{프로모션}` | depth 1에서만 유효. depth 2 이상에서는 depth_1이 프로모션명이 아닌 경우 다수 | **부분 가능** |
| depth_2 = `{페이지명}_{00단}` | depth_2에 위치정보(N단)가 포함된 건은 **0.9%**. 실제로는 노출대상(39.4%)이 주 역할 | **불일치** |
| depth_3 = `{페이지명}_{00단}_{000배너}` | depth_3에 `배너` 포함 건수 **0건**. 노출대상(63.4%)이 주 역할 | **불일치** |

### 3. 표준화를 위한 선행 과제

1. **역할 정의 통일**: 각 depth에 무엇을 넣을지 역할을 먼저 합의해야 함
2. **기존 데이터 마이그레이션**: 현재 데이터는 같은 depth에 서로 다른 의미의 값이 혼재. 자동 변환은 불가하며 규칙+수작업 병행 필요
3. **depth 1 전용 데이터(1,352건)**: 구분자가 없어 분리 자체가 불가능. 신규 등록 시점부터 표준 적용 필요
